# WindowsDoctor KB Governance

Last updated: `2026-04-28`

## 1. Directory Roles
- `knowledge_base/reviewed`: production diagnosis rules.
- `knowledge_base/learned`: learn-only, unreviewed cases generated by `/api/learn`.
- `knowledge_base/archived`: historical tests, duplicate rules, obsolete rules.

## 2. Reviewed Rule Requirements
Reviewed rules should use the standard template:
```powershell
powershell -NoProfile -ExecutionPolicy RemoteSigned -File E:\WindowsDoctor\scripts\New-KBRule.ps1 -Id RULE-ID -Title "Title" -ErrorCode "0x00000000" -Triggers "keyword1","keyword2" -Script "N/A"
```

Required fields:
- `description`
- H1 title
- `EventID/Code`
- `Trigger`
- `Script`
- `## 分析細節`

## 3. Learned Rule Policy
- Learned rules must default to `Script: "N/A"`.
- Learned rules must not be allowlisted directly.
- Learned rules are reviewed manually before promotion.

## 4. Archive Policy
Move rules to `archived` when:
- They are smoke/test cases.
- They duplicate a reviewed rule.
- They contain broad triggers such as `Match`.
- Their remediation is unsafe, stale, or unclear.

## 5. Promotion Checklist
1. Confirm trigger specificity.
2. Confirm diagnosis text is actionable.
3. Confirm repair script exists or keep `Script: "N/A"`.
4. If repairable, create reviewed `Repair-*.bat`.
5. Add script to `scripts/repair-allowlist.json`.
6. Move rule to `reviewed`.
7. Run `scripts\Test-SystemBaseline.ps1`.
