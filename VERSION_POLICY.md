# 版本更新原則

Last updated: `2026-04-28`

## 1. 版本格式
WindowsDoctor 使用三段數字版本：

```text
MAJOR.MINOR.PATCH
```

目前 GUI 顯示可附加預發行標籤，例如：

```text
v0.1.0-alpha
```

## 2. 進位規則
每一段小數點後的數字最大只到 `9`。超過 `9` 必須向前一位進位。

| 目前版本 | PATCH bump | MINOR bump | MAJOR bump |
|---|---|---|---|
| `0.1.0` | `0.1.1` | `0.2.0` | `1.0.0` |
| `0.1.9` | `0.2.0` | `0.2.0` | `1.0.0` |
| `0.9.9` | `1.0.0` | `1.0.0` | `1.0.0` |
| `9.9.9` | `10.0.0` | `10.0.0` | `10.0.0` |

## 3. 升版條件
- `PATCH`: 文件、KB 規則、小型 UI 修正、測試補強、無破壞性修補。
- `MINOR`: 新功能、API 欄位新增、可回溯相容的架構調整。
- `MAJOR`: 破壞性 API/資料格式變更、安全邊界重定義、部署方式不相容。

## 4. 預發行標籤
- `-alpha`: 內部開發、功能可能變動。
- `-beta`: 功能大致穩定，仍需實機驗證。
- `-rc`: 候選發行，僅接受阻斷性修正。

預發行標籤不參與數字進位；進位只套用於 `MAJOR.MINOR.PATCH`。

## 5. 同步位置
升版時必須同步：
- `gui\package.json`
- `gui\src\app\page.tsx`
- `TASK_HANDOFF.md`
- 發行或交接文件中的版本描述

## 6. 禁止事項
- 不允許 `0.1.10`、`0.10.0` 這類小數點後段數超過 `9` 的版本。
- 不允許只改 GUI 顯示版本而不改 `package.json`。
- 不允許在未更新 `TASK_HANDOFF.md` 的情況下視為完成升版。
