狀態意思是否會修復

4. 離線工具自動診斷圖解

自動選用

使用者只輸入問題，系統依分類選出已封裝的 Microsoft 官方診斷工具。

資源控制

工具佇列一次只處理一項；工作視窗顯示 CPU、RAM、程序數與可中斷狀態。

不自動修復

工具輸出會轉成修復報告與 evidence；高風險或未審核項目不會自動修復。

6. 應用場景範例

場景 A：一般 Windows 還能開機

  • 插入 USB，開啟 G:\START_HERE.html。
  • 選擇 GUI-ready 套件，執行啟動器。
  • 先看 Preview，不輸入 RUN。
  • 若有真實錯誤，再保存診斷輸出供日後匯入。

場景 B：Windows 無法正常進桌面

  • 從 USB 開機進 WinPE。
  • 使用離線 KB 查詢故障方向。
  • 只做診斷與預覽，不直接執行修復。
  • 必要時交由人工審查高風險項目。

場景 C：企業 Helpdesk 快速判斷

  • 先跑 USB acceptance，確認套件完整。
  • 用 GUI preview 產生 evidence scoring。
  • safe batch 也必須由操作員輸入 RUN。
  • 將報告交給二線或資安審查。

場景 D：實測後匯入真實錯誤資料

  • 把真實 JSON/log 放入 incoming 對應資料夾。
  • 執行 readiness gate。
  • 只有 PASS 才做 validate/import/rebuild normalized KB。
  • 匯入後重新跑 low-risk baseline 與 USB acceptance。
